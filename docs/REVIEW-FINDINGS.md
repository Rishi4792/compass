# ROUND 3 — third static pass (2026-06-07). Plugin still validates clean; these are design/logic, not mechanical.

> Three fresh agents. They went DEEPER, not smaller — the headline is that round-2's "teeth" are honor-system. Status: OPEN.

## 🔴 CRITICAL
| # | Area | Problem | Fix | Status |
|---|------|---------|-----|--------|
| H1 | receipts/gates (all) | **The receipts+gates are NOT teeth — they're prose relocated into a checkbox block.** The model writes the receipt AND reads it; nothing external verifies a box. The only executable (the hook) just `echo`s advice. So G1 was *renamed*, not fixed; a model under pressure can emit `PASS / [x]` having done nothing and the next stage trusts it. README's "enforcement lives in artifacts" is false as written. | Build a real gate: a **PreToolUse/Stop hook that greps `receipts.md`** and exits non-zero if the required prior receipt is absent / `FAIL` / has a `[ ]`. Commit raw command-output logs the hook asserts exist. Or stop claiming "enforcement" and call it a structured self-checklist. | FIXED |
| H2 | reconciliation (contract/build/review-build) | **Circular reconciliation.** The same model authors the build query AND the "reproducing" query (often identical), and the gold figure can be self-computed by that same query → both agree on the same wrong number → PASS wrong. The flagship check guarantees almost nothing for his flagship use. | Gold figure must be an **INDEPENDENT published number** (audited data-room/MIS/human-signed), pinned as a literal in contract.md — NOT computed by the reproducing query. review-contract fails a self-computed gold. Bake his 3 known bug-classes into the reconcile checklist (dedup / fan-out / source-table). | FIXED |
| H3 | receipts on escalation | **Stale receipts poison the gate.** Receipts are append-only and no re-run invalidates downstream receipts; gates check "a PASS exists" not "the newest, in order." Every escalation loop (build→plan, build→contract) leaves a stale `review-plan PASS` that lets the user skip the re-review the escalation existed to force. | On re-running stage X, mark X's and all later receipts `SUPERSEDED`; gates must read the latest receipt and check it's newer than the prior stage's (monotonic ordering), not just "a PASS somewhere." | FIXED |
